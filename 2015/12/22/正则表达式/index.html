正则表达式的小憩

重要知识点：

分组:     reg1 = /['"][^'"]*['"]/;    reg2 = /(['"])[^'"]*\1/;     reg1 = reg2;
    带圆括号的表达式允许同一正则表达式的后部引用前面的子表达式如reg2的表达式(['"])可以用\1来表示。
    (因为子表达式可以嵌套另一个子表达式，所以它的位置是参与计数的左括号的位置。例如:/([Jj]ava([Ss]cript)?)\sis\s(fun\w*)/ 此式的\2就是([Ss]cript))
    若创建一个不带数字编码的引用，也可以对子表达式进行分组。但此时应用"(?:"和")"。例如:/([Jj]ava(?:[Ss]cript)?)\sis\s(fun\w*)/ 此式的\2就是(fun\w*)。

String支持的4种正则表达式的方法search()、replace()、match()、split()。
    一.search():返回与之匹配的子串的起始位置。若无，返回-1。若search()的参数不是正则表达式，则通过RegExp构造函数将其转换成正则表达式。
           search方法不支持全局检索，因为它会忽略正则表达式参数中的修饰符g(即:全局匹配)
    二.replace():用以执行检索与替换操作。若replace()的第一个参数不是正则表达式，将直接搜索这个字符串。若第一个参数是正则表达式并带有g的话，
则替换所有与正则表达式匹配的元素。否则只替换第一个。
    三.match():返回一个由匹配结果组成的数组。 它的唯一参数是一个正则表达式或者通过RegExp()构造函数将其转换成正则表达式。
    四.split():这个方法用以将调用它的字符串拆分成一个子串组成的数组 它的参数可以是一个正则表达式。
RegExp对象的属性：source、global、ignoreCase、multiline、lastIndex
    source:只读的字符串，包含正则表达式的文本。
    global:只读的布尔值，说明这个正则表达式是否带有修饰符g。
    ignoreCase:只读的布尔值，说明这个正则表达式是否带有修饰符i。
    multiline:只读的布尔值，说明这个正则表达式是否带有修饰符m。
    lastIndex:可读/写的整数。如果匹配模式带有g修饰符，这个属性存储在整个字符串中下一次检索的开始位置。
RegExp对象的exec()、test()方法。
    exec():对一个指定的字符串执行正则表达式。
    test():如果有匹配正则表达式的字符返回true否则返回false。

举例说明:

example1: var email = /^(\w)+(\.\w)*@(\w)+((\.\w{2,3}){1,2})$/  ||  /^(\w)+(\.\w+)*@(\w)+(\w)+((\.\w+)+)$/; 
          //邮箱的两种正则表达式
example2: var url = /^(\w+):\/\/([\w.]+)\/(\S*)$/;  
          //URL的正则表达式写法
          var text = "http://www.baidu.com/~crerich";
          console.log(url.test(text));   //true
example3: var url = /^(\w+):\/\/([\w.]+)\/(\S*)$/;  
          //借用example2的正则表表达式
          var text = "http://www.baidu.com/~crerich";
          console.log(text.match(url));
          运行结果:["http://www.baidu.com/~crerich", "http", "www.baidu.com", "~crerich", index: 0, input: "http://www.baidu.com/~crerich"]
          若此时在url后加个全局匹配g再看下效果
          运行结果:["http://www.baidu.com/~crerich"]
example4: var url = /^(\w+):\/\/([\w.]+)\/(\S*)$/;  
          //同样是上面的url同样是上面的text
          var text = "http://www.baidu.com/~crerich";
          console.log(url.exec(text));
          运行结果:["http://www.baidu.com/~crerich", "http", "www.baidu.com", "~crerich", index: 0, input: "http://www.baidu.com/~crerich"]
          若此时在url后价格全局匹配g我们会发现结果是不变的。
example5: var url = /[c]/g;
          var text = "http://www.baidu.com/~crerich";
          console.log(url.exec(text),url.exec(text))
          运行结果:["c", index: 17, input: "http://www.baidu.com/~crerich"] ["c", index: 22, input: "http://www.baidu.com/~crerich"]
          //注意index的值。

对上述现象做解释:当match方法检索的正则表达式没有设置修饰符g，说明此时match()执行的不是全局搜索，它会返回一个数组。数组的第一个元素就是匹配的字符串，余下的元素则是正则表达式中用圆括号括起来的字表达式。
              index表示检索到第一个字符串的索引，input表示你要检索的字符串。
              exec则不管是否设置修饰符g都会返回跟match不设置修饰符g的效果。此时index表示匹配的第一字符串的索引，再次执行此方法的时候默认从idnex的位置开始搜索字符。因此不同的字符串进行exec搜索的话须置0index。

正则表达式中的直接量字符
字符匹配
\oNUL字符(\u000)
\t制表符
\n换行符
\v垂直制表符
\f换页符
\r回车符
\xnn由十六进制数nn指定的拉丁字符，例如，\xOA等价于\n
\uxxxx由十六进制数xxxx指定的Unicode字符，例如，\u0009等价于\t
\cX控制字符^X，例如，\cJ等价于换行符\n












正则表达式的字符类
[…]方括号内的任意字符
[^…]不在方括号内的任意字符
.除换行符和其他Unicode行终止符之外的任意字符
\w任何ASKII字符组成的单词，等价于[a-zA-Z0-9]
\W任何不是ASKII字符组成的单词，等价于[^a-zA-Z0-9]
\s任何unicode空白符
\S任何非Unicode空白符，注意\w和\S不同
\d任何ASKII数字，等价于[0-9]
\D除了ASKII数字之外的任何字符，等价于[^0-9]
[\b]退格直接量(特例)








正则表达式的重复字符语法
{n,m}匹配前一项至少n次，但不能超过m次
{n,}匹配前一项n次或者更多
{n}匹配前一向n次
?匹配前一项0次或者1次，也就是说前一项是可选的，等价于{0,1}
+匹配前一项1次或者多次，等价于{1,}
*匹配前一项0次或者多次，等价于{0,}








正则表达式中的锚字符
^匹配字符串的开头，在多行检索中，匹配一行的开头
$匹配字符串的结尾，在多行检索中，匹配一行的结尾
\b匹配一个单词的边界，简言之，就是位于字符\w和\W之间的位置，或位于字符\w和字符串的开头或者结尾之间的位置(但需要注意，[\b]匹配的是退格符)
\B匹配非单词边界的位置
(?=p)零宽正向先行断言，要求接下来的字符都与p匹配，但不能包括匹配p的那些字符
(?!p)零宽负向先行断言，要求接下来的字符都不与p匹配





正则表达式的修饰符
i执行不区分大小写的
g执行一个全局匹配
m多行匹配模式






正则表达式的选择、分组和引用字符
|选择，匹配的是该符号左边的子表达式或右边的字表达式
(…)组合，将几个项组合为一个单元，这个单元可通过“*”，“+”，“?”,“|”等符号加以修饰，而且可以记住和这个组合相匹配的字符串以供此后的引用使用
(?:…)只组合，把项组合到一个单元，但不记忆与该组相匹配的字符
\n和第n个分组第一次匹配的字符相匹配，组是圆括号中的子表达式(也有可能是嵌套的)，组索引是从左到右的左括号数，“(?:”形式的分组不编码
