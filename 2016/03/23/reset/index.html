reset

谈谈reset

  • 最近尝试着结合cms来做套完整的网站–映纷视觉。刚准备敲的时候发现自己没有一套完整的reset.css表。于是乎，转向整理一份属于自己的reset表。花了将近一个小时的时间reset1.0出版了。
/*有默认margin以及padding的标签*/
/*ul,ol,p,blockquote*/

/*有默认margin的标签*/
/*body,dl,dd,h1,h2,h3,h4,h5,h6,pre*/

/*有默认padding的标签*/
/*legend*/

/*有默认margin,padding,border的标签*/
/*fieldset*/

/*有默认padding,border的标签*/
/*input,textarea,button,th,td*/

/*有默认border的标签*/
/*table*/

/*需知的一些标签默认属性*/
/*    textarea -> resize属性
 *    改变光标  -> cursor属性
 *     table    -> border-collapse
*/
  • 看到这里你会不会有种轩然一新的feel。我这里对各个标签分的很细跟大多数网上的reset表是完全不同的，这里也不去批判一些那些通用的reset表了。推荐大家看下张鑫旭写的CSS reset的重新审视。
  • 可是当我今天准备再去完善下的时候我发觉我做不下去了。我在想这样一份“庞大”reset表到底适不适合我的网页呢。一个小的站点能用到那么多标签吗？是不是所有标签的自带属性都应该去掉呢？如果去掉设计这些标签时默认样式的意义又是何在呢？……
  • 作为一名前端工程师,我们常把细节、性能优化挂在嘴边。因此从这些角度出发，我更认为自己没有必要写这样一份reset表了。
  • 从 “求经” -> “求精” 的过程中,我们需要审视下：经验固然很好，可与精炼冲突的时候，我们要有选择的能力。
  • 最后再来句我们可以做个lazy boy 然而 clever lazy boy更值得我们去追求。

PC端reset表
body{
    font: 16px/1.5 sans-serif arial "微软雅黑";
    margin: 0;
    }

    p,blockquote{
        margin: 0;
        padding: 0;
    }

    dl,dd,h1,h2,h3,h4,h5,h6,pre{
        margin: 0;
    }

    legend{
        padding: 0;
    }

    fieldset{
        margin: 0;
        padding: 0;
        border: none;
    }

    a{
        text-decoration: none;
    }
    ul,ol{
        margin: 0;
        padding: 0;
        list-style: none;
    }

    /*清除浮动通用方法*/
    .clearfix{
        zoom:1;
    }
    .clearfix:after{
        display: block;
        content: '';
        clear: both;
        overflow: hidden;
        height: 0;
    }
